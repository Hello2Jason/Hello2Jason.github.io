this 总结

JavaScript中的this的指向常常令人感到疑惑，我们来总结一下this指向的规律。

首先我们先说明对于this指向的两个误解。

this既不会指向函数本身也不会指向函数的词法作用域，它的指向是在函数调用时发生绑定，仅与在哪里被调用有关系。

绑定规则：

  1. 默认绑定

    顾名思义，这一绑定规则是在其他规则都不适用的情况下的默认使用规则，当函数直接使用不带任何修饰的函数引用进行调用时，它的this指向全局对象，在浏览器中就为Window，这里要注意，在函数声明在严格模式下的时候，默认绑定中的this指向undefined，并且只有在函数声明中使用严格模式才会有这种情况，当仅在函数调用时使用严格模式的话，this仍然指向Window。

  2. 隐式绑定

    隐式绑定是当函数作为某一对象的方法时，通过该对象的引用访问函数时该函数的this绑定到调用的对象上。同时也要注意，函数的this总是绑定到对象调用链的最后一个对象上（即距离函数最近的对象）。

    另外，隐式绑定时还需要注意，被隐式绑定的对象可能会丢失绑定对象，从而应用默认绑定。

    造成隐式丢失的原因是，新创建的变量y也指向了fn这一函数，然后使用y调用该函数时相当于使用不带任何修饰的函数引用进行调用，所以使用的是默认绑定，函数中的this绑定到了全局变量Window。

    另外，利用JS函数First class function 的特性将函数作为参数传递的时候，也可能发生隐式丢失。

    甚至把函数传入内置函数时的结果也是一样的。

  3. 显式绑定

    显式绑定是指通过使用JS提供的call()与apply()函数来显示地设置某一个函数的this所绑定的对象，这两个函数的区别在于，向绑定的函数传递参数的形式不同，call()直接传递单个变量，apply()传递一个数组，并将数组中每一项对应一个参数传递给函数。

    通过这两种方法给函数绑定的绑定this对象还是可以被修改，因此我们可以采取一种叫做硬绑定的方法

  4. new绑定
    new绑定就是当我们对某一函数使用new操作符时，函数体中的this会被绑定到刚实例化的一个新对象上。
    new操作符实例化对象时进行的操作：

    1. 创建一个新的对象
2. 将新对象的原型（__proto__）指向构造函数的原型对象（.prototype）
3. 将函数体内的this指向新创建的对象并执行函数体内的程序
4. 如果函数没有返回一个对象则返回新创建的对象，否则返回指定对象

  5. 优先级
    以上规则就是会影响this绑定的所有情况，然后他们有一个优先级，保证在多种情况同时发生的情况下可以确定this的指向，他们的优先级是：

    new绑定 > 显示绑定 > 隐式绑定 > 默认绑定
  6. 一些例外情况：
    • 当把null或者undefined传入call, apply, bind, 绑定后，函数会忽略他们然后采用默认绑定this绑定到全局对象
    • (p.foo = o.foo)() 这种情况会采用默认绑定
  7. ES6箭头函数的this
    ES6的箭头函数中的this的绑定机制与普通函数完全不同，它的this绑定机制更像是作用域链的词法作用域。

闭包总结

JavaScript中的闭包，一般的定义是：有权访问另一个函数作用域中变量的函数 ，这个定义说得不是很清楚，接下来用例子来详细解释。

我们都知道JS中没有快级作用域，能够建立新作用域的方法，只有新建函数，在一个函数内部的变量是属于这个函数的作用域的，在它外部的作用域中无法直接访问到它当中的变量，但是这个函数当中却有权访问它声明时候的外部作用域中的变量，这个是因为JS的作用域链这个机制：每一个函数都有权访问它声明位置外部的作用域内的变量 。

上面一段代码演示了函数作用域以及作用域链，有一点需要注意的是，当不使用严格模式时，如果定义变量之前不加var，这个变量就会被定义为全局变量，所有的作用域中都可以访问到。

在JS中有很多时候都需要使用另一个函数作用域中的变量，这时候就要使用闭包了，按照之前的我们知道函数作用域链，在函数内部声明的函数可以访问到该函数作用域中的变量，我们如果将这个内部的函数返回给最外部作用域中的一个变量，就可以通过这个变量指向的函数来访问它原本访问不到的函数作用域中的变量了。

以上描述之所以能够实现，还有一个重要的因素，那就是JS中的函数不仅是对象，同时也是一级函数(First Class Function)，一级函数的意思就是你可以对这个函数作与所有其他变量能做的操作，比如将函数作为一个函数的返回值，或者将它作为参数传递给一个函数等。

虽然最后访问到了i，但是我们可以看到，结果与我们所期望的似乎不同，最后返回的数组中每一个函数执行的结果都是返回3，这个就要从闭包产生的过程来解释了，首先当以上JS文件加载完成后，会先建立全局执行环境，在创建阶段将声明的函数以及变量名称保存在内存中，执行阶段一行一行执行代码，执行到调用createFn()时，创建createFn函数的执行环境，同样的创建阶段保存声明，执行阶段执行函数内的代码，执行完成后返回arr，然后清除调这个函数的执行环境，清除执行环境后，本来里面的变量以及函数都将被垃圾回收机制清理，但是由于全局变量中的变量fn引用了arr，于是arr以及其中用到的变量i都不会被清理掉仍保存在内存空间中，然后到了执行数组中每一个函数的时候首先会创建函数的执行环境，然后执行函数时，发现函数中会用到变量i，由于变量i对于执行的函数使能够通过作用域链访问到的，变量i就存在于函数fn的闭包中，能够访问到，然后因为执行fn时变量i的值为之前执行玩createFn后最后保存的值3，于是就会返回3，数组中的另外两个函数也同理，它们都是访问的同一个内存中的变量i，值都相等为3。

深入理解CSS：padding

  1. padding与元素的尺寸

    对于块级(block)元素

    padding会影响元素的尺寸；但是当width:auto或者box-sizing:border-box;时，padding不会影响元素的尺寸(有例外！)，只会压缩内容区域的尺寸，但是如果padding的尺寸过大时，也会影响元素尺寸。

    总结：

    1. padding值暴走，一定会影响元素尺寸
    2. width非auto并且box-sizing非border-box时，padding影响尺寸
    3. width为auto或box-sizing为border-box，同时padding值没有暴走，不影响尺寸

    对于内联(inline)元素

    水平padding影响元素尺寸，垂直padding不影响尺寸，但是水平和垂直padding都会影响元素的背景区域大小，会影响scrollHeight。

    应用：

    实现高度可控的分隔线

JavaScript 执行环境

​ 在JS中，每当一个JS文件被加载执行后，首先做的事情是创建一个全局执行环境(Global Execution Environment)，在这个全局执行环境中会先生成一个变量对象(Variable Environment)、一个”this”指针(在浏览器中指向Window对象)、一个指向原型链的Outer Environment。

​ 然后创建执行环境分为两个阶段，第一个创建阶段的主要任务是，查看整个文档的代码，当发现变量或者函数声明语句后，在这个阶段会先建立这些变量以及函数在内存中的存放位置，同时如果是变量声明的话，这个阶段会将变量的值先设置为“undefined”，如果是函数声明的话，则会将整个函数的代码都存入内存中(函数提升)；然后第二个阶段称为执行阶段，在这个阶段中JS会一行一行地执行文件中的代码，这个时候如果之前声明的变量有赋值的话，才会对他进行赋值操作。
